Using Multiple Custom Domains With Github Pages

26 Jun 2020, 16:46

When setting up this blog I decided I wanted to use a different domain than the main one I already had. I spent quite a bit of time trying to figure out if it was possible to use GitHub Pages to use multiple custom domains for multiple repositories and exactly how to do it.

I finally found this post which set me straight on both counts. Here are both the steps outlined again and a non-exhaustive list of the things I bumped into but ultimately were not part of the solution.

Solution

TL;DR Github handles the routing to the custom domain from <username>.github.io. for all of your repos.

  1. Setup CNAME record with your DNS provider. Host is www and Value as <username>.github.io.
    • Yes, include the trailing dot
    • No, don’t include anything else like <username>.github.io./<repo-name>/
  2. Add A records for the GitHub Pages IP addresses:
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
  1. Set the custom domain on the GitHub repo

This will work for as many repos and domains as you want.

Not Solutions

These are some things I encountered on StackOverflow and various other blog pages as required steps to get this to work but ultimately were not required.

  • No special branch names (gh-pages).
    • I deploy from the master branch for both repositories. No special branch names required.
  • No weird DNS redirecting records.
    • Some solutions suggested using various redirect records with you DNS provider. I only required the A records for the GitHub Pages IPs and a CNAME record for the domain

GitHub Pages Options

During my searching and testing I found other folks trying to do similar things with GitHub Pages. There are 3 options from what I’ve gathered:

  1. Single GH repo -> Multiple custom domains
  2. Multiple GH repos -> Single custom domain
  3. Multipl GH repos -> A single Custom Domain for each repo

I’m not sure if #1 is currently possible without duplicating the repository. Theoretically you could clone the repo for as many domains as you require but it’s not a happy-path solution by any means.

Option #2 is really straightforward! Setting up a single custom domain for a repo named <username>.github.io allows you to turn on GH Pages for any other public repository in your account and it will be available at custom-domain.com/repo-name/ automatically.

Option #3 is the goal I was trying to achieve and you can see it’s really simple even though the documentation is a little unclear.
